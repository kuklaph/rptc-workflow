Research Report: Example Template

Research Mode: Comprehensive Status: Complete Version: 1.0.0 Date: 2025-10-20 Author: Claude

Overview

This is a comprehensive HTML template demonstrating professional dark-themed report styling. It serves as a reference for generating HTML reports from markdown research documents.

The template includes all required features: responsive design, table of contents, syntax highlighting, reading progress indicator, and accessibility support.

Executive Summary

Key Insight: This template demonstrates a complete, production-ready HTML structure with embedded CSS and JavaScript. All interactive features are fully functional and can be opened directly in any modern browser.

The template follows industry-standard practices for professional documentation:

  • GitHub Dark color palette - WCAG AA compliant contrast ratios
  • Responsive design - Works on desktop, tablet, and mobile devices
  • Accessibility features - ARIA labels, semantic HTML, keyboard navigation
  • Progressive enhancement - Core content accessible without JavaScript

Key Findings

Finding 1: Architecture

The template uses a semantic HTML5 structure with proper landmark roles for accessibility:

  • <nav> for table of contents sidebar
  • <main> for primary content area
  • <header>, <section>, <article> for content organization
  • ARIA labels for interactive elements (progress bar, buttons, navigation)

Finding 2: Performance

Performance optimizations include:

  1. Embedded CSS and JavaScript (single HTTP request)
  2. Minimal external dependencies (only Prism.js for syntax highlighting)
  3. CSS transitions instead of JavaScript animations
  4. Efficient scroll event handling with throttling

Finding 3: Security

Important: When generating HTML from user-provided markdown, always sanitize input to prevent XSS attacks. Escape HTML entities and validate URLs before rendering.

Python Example

Python 
def generate_html_report(markdown_content, title="Research Report"):
    """
    Convert markdown content to HTML report with dark theme.

    Args:
        markdown_content: Source markdown text
        title: Report title

    Returns:
        Complete HTML document as string
    """
    # Parse markdown to HTML
    html_body = markdown.markdown(markdown_content)

    # Generate table of contents from headings
    toc = extract_headings(html_body)

    # Apply template with embedded styles
    return render_template(
        'report-structure.html',
        title=title,
        content=html_body,
        toc=toc
    )

JavaScript Example

JavaScript 
// Reading progress bar implementation
function updateProgressBar() {
    const windowHeight = window.innerHeight;
    const documentHeight = document.documentElement.scrollHeight;
    const scrollTop = window.pageYOffset || document.documentElement.scrollTop;

    const scrollPercent = (scrollTop / (documentHeight - windowHeight)) * 100;

    const progressBar = document.querySelector('.progress-bar-fill');
    progressBar.style.width = scrollPercent + '%';
}

// Throttle scroll events for performance
window.addEventListener('scroll', throttle(updateProgressBar, 16));

Bash Example

Bash 
#!/bin/bash
# Generate HTML report from markdown

INPUT_FILE="research.md"
OUTPUT_FILE="research-report.html"

echo "Converting $INPUT_FILE to HTML..."

# Convert markdown to HTML
pandoc "$INPUT_FILE" \
    --template=report-structure.html \
    --standalone \
    --toc \
    --output="$OUTPUT_FILE"

echo "Report generated: $OUTPUT_FILE"

Best Practices

  1. Semantic HTML: Use proper heading hierarchy (h1 → h2 → h3) for better accessibility
  2. Descriptive IDs: Generate IDs from heading text (lowercase, hyphenated) for reliable anchor links
  3. Alt text: Always include alt attributes for images
  4. External links: Add target="_blank" rel="noopener noreferrer" for security
  5. Mobile testing: Test on actual devices, not just browser dev tools
